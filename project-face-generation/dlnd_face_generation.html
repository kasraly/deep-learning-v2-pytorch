Face Generation¶

In this project, you'll define and train a DCGAN on a dataset of faces. Your goal is to get a generator network to generate new images of faces that look as realistic as possible!

The project will be broken down into a series of tasks from loading in data to defining and training adversarial networks. At the end of the notebook, you'll be able to visualize the results of your trained Generator to see how it performs; your generated samples should look like fairly realistic faces with small amounts of noise.

class Discriminator(nn.Module):

    def __init__(self, conv_dim):
        """
        Initialize the Discriminator Module
        :param conv_dim: The depth of the first convolutional layer
        """
        super(Discriminator, self).__init__()
        self.conv_dim = conv_dim
        # complete init function
        layers = [
            nn.Conv2d(3, conv_dim, kernel_size=3, stride=1, padding=1, bias=True),  # 32x32
            nn.LeakyReLU(0.1),
            nn.Conv2d(conv_dim, conv_dim*2, kernel_size=6, stride=2, padding=2, bias=False), #16x16
            nn.BatchNorm2d(conv_dim*2),
            nn.LeakyReLU(0.1),
            nn.Conv2d(conv_dim*2, conv_dim*2, kernel_size=5, stride=1, padding=2, bias=False), #16x16
            nn.BatchNorm2d(conv_dim*2),
            nn.LeakyReLU(0.1),
            nn.Conv2d(conv_dim*2, conv_dim*4, kernel_size=4, stride=2, padding=1, bias=False), #8x8
            nn.BatchNorm2d(conv_dim*4),
            nn.LeakyReLU(0.1),
            nn.Conv2d(conv_dim*4, conv_dim*4, kernel_size=4, stride=2, padding=1, bias=False), #4x4
            nn.BatchNorm2d(conv_dim*4),
            nn.LeakyReLU(0.1),
        ]
        self.conv_layers = nn.Sequential(*layers)
        self.linear = nn.Linear(conv_dim*4*4*4, 1)

    def forward(self, x):
        """
        Forward propagation of the neural network
        :param x: The input to the neural network     
        :return: Discriminator logits; the output of the neural network
        """
        # define feedforward behavior
        x = self.conv_layers(x)
        x = x.view(-1, self.conv_dim*4*4*4)
        x = self.linear(x)
               
        return x

Generator¶

The generator should upsample an input and generate a new image of the same size as our training data 32x32x3. This should be mostly transpose convolutional layers with normalization applied to the outputs.

Exercise: Complete the Generator class¶

  • The inputs to the generator are vectors of some length z_size
  • The output should be a image of shape 32x32x3
In [11]:
class Generator(nn.Module):
    
    def __init__(self, z_size, conv_dim):
        """
        Initialize the Generator Module
        :param z_size: The length of the input latent vector, z
        :param conv_dim: The depth of the inputs to the *last* transpose convolutional layer
        """
        super(Generator, self).__init__()
        self.conv_dim = conv_dim
        
        # first, fully-connected layer
        self.linear = nn.Linear(z_size, conv_dim*4*4*4, bias=True) # 4x4

        layers = [
            nn.BatchNorm2d(conv_dim*4),
            nn.LeakyReLU(0.1),
            nn.ConvTranspose2d(conv_dim*4, conv_dim*2, kernel_size=4, stride=2, padding=1, bias=False),  # 8x8
            nn.BatchNorm2d(conv_dim*2),
            nn.LeakyReLU(0.1),
            nn.ConvTranspose2d(conv_dim*2, conv_dim*2, kernel_size=4, stride=2, padding=1, bias=False),  # 16x16
            nn.BatchNorm2d(conv_dim*2),
            nn.LeakyReLU(0.1),
            nn.ConvTranspose2d(conv_dim*2, conv_dim, kernel_size=5, stride=1, padding=2, bias=False), #16x16
            nn.BatchNorm2d(conv_dim),
            nn.LeakyReLU(0.1),
            nn.ConvTranspose2d(conv_dim, conv_dim, kernel_size=6, stride=2, padding=2, bias=False), #32x32
            nn.LeakyReLU(0.1),
            nn.ConvTranspose2d(conv_dim, 3, kernel_size=3, stride=1, padding=1, bias=True), #32x32
            nn.Tanh(),
        ]
        self.deconv_layers = nn.Sequential(*layers)
       
    def forward(self, x):
        """
        Forward propagation of the neural network
        :param x: The input to the neural network     
        :return: A 32x32x3 Tensor image as output
        """
        # define feedforward behavior
        x = self.linear(x)
        x = x.view(-1, self.conv_dim*4, 4, 4) # (batch_size, depth, 4, 4)
        x = self.deconv_layers(x)
               
        return x

Discriminator and Generator Losses¶

Now we need to calculate the losses for both types of adversarial networks.

Discriminator Losses¶

  • For the discriminator, the total loss is the sum of the losses for real and fake images, d_loss = d_real_loss + d_fake_loss.
  • Remember that we want the discriminator to output 1 for real images and 0 for fake images, so we need to set up the losses to reflect that.

Generator Loss¶

The generator loss will look similar only with flipped labels. The generator's goal is to get the discriminator to think its generated images are real.

Exercise: Complete real and fake loss functions¶

You may choose to use either cross entropy or a least squares error loss to complete the following real_loss and fake_loss functions.

def real_loss(D_out):
    '''Calculates how close discriminator outputs are to being real.
       param, D_out: discriminator logits
       return: real loss'''
    device = torch.device('cuda' if train_on_gpu else 'cpu')
    labels = torch.zeros_like(D_out, device=device) + 0.9
    criterion = nn.BCEWithLogitsLoss()
    loss = criterion(D_out, labels)
    return loss

def fake_loss(D_out):
    '''Calculates how close discriminator outputs are to being fake.
       param, D_out: discriminator logits
       return: fake loss'''
    device = torch.device('cuda' if train_on_gpu else 'cpu')
    labels = torch.zeros_like(D_out, device=device)
    criterion = nn.BCEWithLogitsLoss()
    loss = criterion(D_out, labels)
    return loss

def train(D, G, n_epochs, print_every=50):
    '''Trains adversarial networks for some number of epochs
       param, D: the discriminator network
       param, G: the generator network
       param, n_epochs: number of epochs to train for
       param, print_every: when to print and record the models' losses
       return: D and G losses'''
    
    # move models to GPU
    if train_on_gpu:
        D.cuda()
        G.cuda()

    # keep track of loss and generated, "fake" samples
    samples = []
    losses = []

    # Get some fixed data for sampling. These are images that are held
    # constant throughout training, and allow us to inspect the model's performance
    sample_size=16
    fixed_z = np.random.uniform(-1, 1, size=(sample_size, z_size))
    fixed_z = torch.from_numpy(fixed_z).float()
    # move z to GPU if available
    if train_on_gpu:
        fixed_z = fixed_z.cuda()

    # epoch training loop
    for epoch in range(n_epochs):

        # batch training loop
        for batch_i, (real_images, _) in enumerate(celeba_train_loader):

            batch_size = real_images.size(0)
            real_images = scale(real_images)

            # ===============================================
            #         YOUR CODE HERE: TRAIN THE NETWORKS
            # ===============================================
            
            # 1. Train the discriminator on real and fake images
            d_optimizer.zero_grad()           
            if train_on_gpu:
                real_images = real_images.cuda()
            
            D_real = D(real_images)
            d_real_loss = real_loss(D_real)

            z = np.random.uniform(-1, 1, size=(batch_size, z_size))
            z = torch.from_numpy(z).float()
            # move x to GPU, if available
            if train_on_gpu:
                z = z.cuda()
            fake_images = G(z)
            
            # Compute the discriminator losses on fake images            
            D_fake = D(fake_images)
            d_fake_loss = fake_loss(D_fake)
            
            # add up loss and perform backprop
            d_loss = d_real_loss + d_fake_loss
            d_loss.backward()
            d_optimizer.step()

            # 2. Train the generator with an adversarial loss
            g_optimizer.zero_grad()
            
            # 1. Train with fake images and flipped labels
            
            # Generate fake images
            z = np.random.uniform(-1, 1, size=(batch_size, z_size))
            z = torch.from_numpy(z).float()
            if train_on_gpu:
                z = z.cuda()
            fake_images = G(z)
            
            # Compute the discriminator losses on fake images 
            # using flipped labels!
            D_fake = D(fake_images)
            g_loss = real_loss(D_fake) # use real loss to flip labels
            
            # perform backprop
            g_loss.backward()
            g_optimizer.step()           
            
            # ===============================================
            #              END OF YOUR CODE
            # ===============================================

            # Print some loss stats
            if batch_i % print_every == 0:
                # append discriminator loss and generator loss
                losses.append((d_loss.item(), g_loss.item()))
                # print discriminator and generator loss
                print('Epoch [{:5d}/{:5d}] | d_loss: {:6.4f} | g_loss: {:6.4f}'.format(
                        epoch+1, n_epochs, d_loss.item(), g_loss.item()))


        ## AFTER EACH EPOCH##    
        # this code assumes your generator is named G, feel free to change the name
        # generate and save sample, fake images
        G.eval() # for generating samples
        samples_z = G(fixed_z)
        samples.append(samples_z)
        G.train() # back to training mode

    # Save training generator samples
    with open('train_samples.pkl', 'wb') as f:
        pkl.dump(samples, f)
    
    # finally return losses
    return losses

training process¶

I started the experiment with small models (conv_dim of 4 for both D and G). The generator learned to created some blurry images with a face-shaped circle in the middle, but the face was not recognizable.

I increased the model capcaity (conv_dim of 16 for both D and G), and noticed the much improved faces, however, the descrimitor loss was much lower compared to generator.

I feel descriminator has an easier job compared to generator; therefore, the generator capacity should be higher than descriminator. I increased the generator model size (conv_dim of 16 for both D and and 64 for G).

The descriminator is the usual supervised learning with the exception of data for one class being changed at every step. The generator, however, is learning a moving target. The generator learns from the way the descriminator differentiates between real and fake images, and tries to replicate the features that descriminator uses for classification. In the next try I decreased the learning speed of the generator compared descrimintor. This way discriminator is expected to proeprly idenitfy the real and fake images, hence helping the generator to create more realistic images. This initially worked quite well, but seemed to cause oscillation in the training of generator. Possibly, descrimintor was too good at finding fake images and causing the generator to switch the face style dramatically.

My last try was to reduce the learning speed of the descriminator compared to generator. This way the moving target for generator which is the descriminator, would learn slower than generator, resulting is more smooth and stable learning for generator. The downside was slower learning of the generator as the descriminator was not providing well guidance for generator.

Generated Samples¶

Looking at the progress of genereted samples over the training process, the competition between generator and descriminator is clear. For example, some generated samples start proper eyes, then eyes changed to dark circle (resembling very dark sunglasses), and later changing to lighter sunglasses (with eye visible from behind shade) . Probably it is a way for the descriminator to identify the fake and real images, but the generator will learn to add glassess properly and overcome the descrimintaor.

The biggest challenge with the training is the balance between training descrimintor and generator. Having a very aggressive descrimintor might cause oscillatory behaviour in the generator. On the hand a weak descrimintor cannot properly differentiate between real and fake images and does provide effective learning information for generator.

The generated samples mostly have light faces, which correlates with that fact that most faces are from white people. However, some generated faces have attributes that are less common in the dataset, such as hats, glasses, and darker skin colors.
